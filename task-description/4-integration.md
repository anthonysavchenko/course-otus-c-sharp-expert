# 4. Интеграция и первая рабочая версия: "Все вместе"

**Лекция 15.06.2026. Lock-Free алгоритмы с Interlocked и Volatile**

## Цель

- Интегрировать независимые компоненты (сетевой уровень, парсер, хранилище) в единое работающее приложение.
- Реализовать полный цикл обработки запроса: получение данных, парсинг, выполнение логики, формирование и отправка ответа.
- Получить практический опыт в преобразовании данных для сетевой передачи (string, byte[], Span).
- Создать базовую, но полнофункциональную версию продукта, которая станет основой для дальнейших улучшений.

## Результат

Студент получит работающее консольное приложение — TCP-сервер, который можно запустить и протестировать с помощью сетевого клиента (например, telnet). Сервер будет корректно обрабатывать команды SET, GET и DELETE, взаимодействуя с потокобезопасным хранилищем данных.

## Зачем это нужно

Это задание — ключевой момент курса, где разрозненные части кода превращаются в осязаемый, работающий продукт. Оно закрепляет понимание клиент-серверной архитектуры и дает студенту законченное приложение, которое можно профилировать, тестировать под нагрузкой и улучшать в последующих домашних заданиях.

## Описание/Пошаговая инструкция выполнения домашнего задания

Пункт 1. Интеграция SimpleStore в TcpServer
Возьмите класс TcpServer из ДЗ №2 и класс SimpleStore из ДЗ №3. Модифицируйте TcpServer так, чтобы он создавал и хранил один экземпляр SimpleStore. Лучше всего передавать экземпляр хранилища через конструктор сервера.

Пункт 2. Реализация логики обработки команд
В методе ProcessClientAsync после парсинга команды добавьте блок switch (или if/else if), который будет определять, какую операцию выполнить. Внутри этого блока вызывайте соответствующие методы вашего экземпляра SimpleStore (Set, Get или Delete). Не забудьте корректно преобразовывать данные из ReadOnlySpan в string для ключей и в byte[] для значений (span.ToArray()).

Пункт 3. Формирование и отправка ответа для SET и DELETE
После успешного выполнения команд SET и DELETE сервер должен отправить клиенту подтверждение. Сформируйте простой ответ, например, строку "OK\r\n". Преобразуйте эту строку в byte[] (Encoding.UTF8.GetBytes) и отправьте клиенту с помощью await clientSocket.SendAsync(...).

Пункт 4. Формирование и отправка ответа для GET
После выполнения команды GET возможны два исхода. Если \_store.Get(key) вернул значение (byte[]), отправьте эти байты клиенту. Если метод вернул null (ключ не найден), отправьте клиенту стандартизированный ответ, например, строку "(nil)\r\n".

Пункт 5. Обработка некорректных команд
Добавьте в блок switch ветку default для обработки всех неизвестных команд. В этом случае сервер не должен падать, а должен отправить клиенту сообщение об ошибке, например, "-ERR Unknown command\r\n".

## Критерии оценки

Экземпляр SimpleStore корректно создается и используется внутри TcpServer - 1 балл
Реализована логика для команд SET и DELETE, включая вызов методов хранилища и отправку "OK" ответа клиенту - 3 балла
Реализована логика для команды GET, включая отправку найденного значения или стандартизированного "null" ответа - 3 балла
Сервер корректно обрабатывает неизвестные команды, отправляя клиенту сообщение об ошибке, и не падает - 2 балла
Все необходимые преобразования типов данных (Span в string, Span в byte[], string в byte[]) выполнены корректно - 1 балл
