# 3. Потокобезопасное хранилище и сбор статистики

**Лекция 04.06.2026. Каналы (System.Threading.Channels): современный асинхронный producer/consumer**

## Цель

- Применить на практике продвинутые примитивы синхронизации для создания высокопроизводительных потокобезопасных структур данных.
- Освоить lock-free техники для сбора метрик в многопоточной среде, избегая узких мест производительности.
- Понять разницу в производительности между блокирующими и lock-free подходами при решении разных задач.

## Результат

Студент получит модифицированный класс SimpleStore из ДЗ №1, который теперь можно безопасно использовать из нескольких потоков одновременно. Этот класс также будет предоставлять публичные свойства для получения статистики своей работы (количество операций).

## Зачем это нужно

Любой серверный сервис по своей природе является многопоточным. Умение правильно защищать разделяемые данные — это фундаментальный навык. Это ДЗ учит использовать не просто lock, а более тонкий и производительный инструмент ReaderWriterLockSlim, а также показывает, как избежать блокировок вообще с помощью Interlocked там, где это возможно, что является признаком экспертного подхода.

## Описание/Пошаговая инструкция выполнения домашнего задания

Пункт 1. Возьмите класс SimpleStore из ДЗ №1. Добавьте в него приватное поле private readonly ReaderWriterLockSlim \_lock = new ReaderWriterLockSlim();. Модифицируйте методы Set, Get и Delete так, чтобы они использовали этот лок. Get должен использовать блокировку на чтение (EnterReadLock/ExitReadLock), а Set и Delete — блокировку на запись (EnterWriteLock/ExitWriteLock). Обязательно используйте конструкцию try...finally для гарантированного освобождения блокировки.

Пункт 2. Добавление полей для статистики
Добавьте в SimpleStore три приватных long поля для хранения статистики: \_setCount, \_getCount, \_deleteCount. Создайте публичный метод или свойство (например, GetStatistics()), которое будет возвращать кортеж (long setCount, long getCount, long deleteCount).

Пункт 3. Реализация сбора статистики
Модифицируйте методы Set, Get и Delete. Внутри каждого метода после успешного выполнения основной операции инкрементируйте соответствующий счетчик. Используйте Interlocked.Increment(ref \_fieldName) для потокобезопасного увеличения счетчиков.

Пункт 4. Реализация IDisposable
Реализуйте в классе SimpleStore интерфейс IDisposable. В методе Dispose освободите ресурсы, занимаемые ReaderWriterLockSlim (вызвать его метод Dispose).

Пункт 5. Написание тестов на многопоточность
Создайте новый юнит-тест (или добавьте в существующий). Напишите тест, который запускает несколько задач (Task.Run) параллельно. Часть задач должна вызывать метод Set, а другая часть — Get. Тест должен проверить, что после выполнения всех задач данные в хранилище корректны и не произошло гонки состояний, а также что счетчики статистики показывают правильные значения.

## Критерии оценки

Метод Get использует ReaderWriterLockSlim в режиме чтения (EnterReadLock/ExitReadLock) - 2 балла
Методы Set и Delete используют ReaderWriterLockSlim в режиме записи (EnterWriteLock/ExitWriteLock), блокировки корректно освобождаются в блоке finally - 3 балла
Реализован потокобезопасный сбор статистики с использованием Interlocked.Increment для трех операций - 2 балла
Класс SimpleStore реализует интерфейс IDisposable и корректно освобождает ресурсы ReaderWriterLockSlim - 1 балл
Написан многопоточный тест, который доказывает корректность работы хранилища и сбора статистики под нагрузкой - 2 балла
